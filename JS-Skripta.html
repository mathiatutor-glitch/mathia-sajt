1 Funkcije, scope i closure

Funkcije pišeš klasično ili kao strelaste. Scope je gde promenljiva važi. Closure je funkcija koja pamti svoje okruženje.

const saberi = (a, b) => a + b; strelasta function brojac() { let n = 0; return () => ++n; closure: pamti n } const c = brojac(); c(); c(); 1, pa 2
Rešen primer

brojac() vraća funkciju koja pamti n. Svaki poziv c() uvećava isti n — to je closure. ✔️

Ada kaže

let i const imaju blok-scope (važe unutar {}). Izbegavaj var — pravi suptilne greške. 🌟

Napiši strelastu funkciju koja vraća kvadrat.

const kv = x => x*x;

2 Objekti, klase, prototipovi

Objekat su parovi ključ:vrednost. Klasa je šablon za objekte.

const osoba = { ime: 'Ana', godine: 20 }; osoba.ime; 'Ana' class Pas { constructor(ime){ this.ime = ime; } laji(){ return this.ime + ' laje'; } } const p = new Pas('Reks');
Rešen primer

new Pas('Reks') pravi objekat; p.laji() vraća „Reks laje". this pokazuje na objekat. ✔️

Ada kaže

this zavisi od toga kako je funkcija pozvana. Strelaste funkcije ne menjaju this — zgodne u metodama. 💡

Dodaj objektu osoba polje grad = „Niš".

osoba.grad = 'Niš';

3 Metode nizova (map / filter / reduce)

map transformiše svaki element, filter zadrži one koji prolaze uslov, reduce sažme u jednu vrednost.

const a = [1, 2, 3, 4]; a.map(x => x*2); [2,4,6,8] a.filter(x => x % 2 === 0); [2,4] a.reduce((z, x) => z + x, 0); 10 (zbir)
Rešen primer

a.filter(x=>x>2).map(x=>x*10) → [30, 40]. Metode se lančaju. ✔️

Ada kaže

map i filter prave novi niz (ne menjaju original) — zato su sigurni i čitljivi. 🌟

Od [1,2,3,4,5] uzmi parne i udvostruči.

a.filter(x=>x%2===0).map(x=>x*2).

4 Destrukturiranje, spread / rest

Destrukturiranje izvlači vrednosti iz niza/objekta. Spread (...) širi, rest (...) skuplja.

const [a, b] = [1, 2]; a=1, b=2 const { ime } = osoba; ime iz objekta const spojeni = [...n1, ...n2]; spread function f(...args) { } rest: sve u niz
Rešen primer

const {ime, godine} = osoba izvlači oba polja odjednom. [...[1,2], ...[3,4]] = [1,2,3,4]. ✔️

Ada kaže

Spread pravi plitku kopiju: const kopija = [...niz] — korisno da ne menjaš original. 💡

Zameni vrednosti a i b jednom linijom.

[a, b] = [b, a];

5 Asinhroni JS (Promise, async / await)

JS ne čeka spore stvari (mreža) — koristi Promise. async/await piše asinhrono kao da je sinhrono.

async function uzmi() { try { const odg = await fetch(url); const data = await odg.json(); return data; } catch (e) { console.error(e); } }
Rešen primer

await pauzira funkciju dok Promise ne završi, pa nastavi sa rezultatom. try/catch hvata grešku. ✔️

Ada kaže

await sme samo unutar async funkcije. Zaboraviš li async, dobićeš grešku. 🌟

Šta async funkcija uvek vraća?

Promise.

6 fetch i moduli

fetch dohvata podatke sa mreže (vraća Promise). Moduli dele kod u fajlove sa export/import.

mat.js export function saberi(a, b){ return a + b; } main.js import { saberi } from './mat.js'; fetch('/api/podaci') .then(r => r.json()) .then(d => console.log(d));
Rešen primer

export deli funkciju, import je uvozi. fetch vraća Promise koji razrešiš sa .then ili await. ✔️

Ada kaže

fetch ne baca grešku na 404 — proveri odg.ok pre čitanja. Česta zamka. 💡

Kako uvoziš funkciju saberi iz mat.js?

import { saberi } from './mat.js';
